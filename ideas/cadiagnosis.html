Training with patches

Question:

How does inference work when I train a neural network on patches?

Replies:

  • M: Check the sliding_window_inference from MONAI. It scans the image according to your patch size defined (roi_size), performs predictions on each individually, and combines the projections at the end.
  • S: Tensors should solve this problem, considering every image is a tensor of cropped images. Crop the image into parts and then put them as a batch through the network.
  • J: You can create a new model with the new input size, load the weights, and run.

Feature extraction

Question:

I am currently trying to understand feature extraction for kmeans clustering of images. I am using for example 32 gabor filters - this means every image (64x64) gets 32 filtered versions. Should I:

  1. generate a feature vector (646432=131072) by just concatenating those?
  2. generate a feature vector (64*64=4096) by somehow combining the pixels?
  3. generate one feature for each filter (=32)?
  4. do something else?

Replies:

  • V: The standard thing was to make histograms per filter. So eg 32 filters x 10 bins for 320 features. This way images don't need to be the same size. Create a histogram, then concatenate the histograms. This is an example that I worked on a couple of years ago: https://ieeexplore.ieee.org/document/8094884 . These filters are similar to Gabor filters and I used 8 filters x 4 scales x 10 bins = 320 features. Use adaptive binning for the bins. If you just quantize the [min, max] interval for a filter into bins, you might have a bin that's very full, and all other bins are empty. So the bin size should be varied, so that each of the bins is a bit informative. You can do this by, for a single filter, taking a subset (e.g. 10) of the images filtered with that filter, concatenating them (so in essense you are creating a big image), and then looking at the quantiles of the pixel distributions.
  • S: In any case, after applying Gabor filters to the image you'll want to look into methods to combine them (not just concatenating) such as Local Energy, Mean Amplitude, Phase Amplitude... your feature vectors will have to be low-dimensional (e.g., 1x32) and then you'll apply the clustering.

PCA

Question:

How can we use PCA after pre-processing?

Replies:

  • S: Pre-processing (normalization, scaling, resizing) should not affect the nature of the data, so typically it's okay to use PCA for representing the data in a lower dimensional space after the pre-processing.
